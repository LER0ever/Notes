    Math240-04 Propositional Equivalences

    System Specification

    Consistency of a collection of specifications.

    X >= 1 and X <= 5
    X >= 1 and X <= 0

    System specifications should be consistent, that is, they should not contain conflicting requirements that could be used to derive a contradiction. When specifications are not consistent, there would be no way to develop a system that satisfies all specifications.

    Ex.
    p ∨ ¬q : “The diagnostic message is stored in the buffer or it is retransmitted.”
    ¬q : “The diagnostic message is not stored in the buffer.”
    p → q : “If the diagnostic :-: |message is stored in the buffer, then it is retransmitted.”

    pqp ∨ q¬qp → q
    FTTTT

    Equivalences

    Two compound propositions are equivalent if they have the same truth value no matter what.

    p → q and ¬q → ¬p are equivalent

    A compound proposition that is always true, no matter what the truth values of the proposi- tional variables that occur in it, is called a tautology.

    p ∨ ¬p

    A compound proposition that is always false is called a contradiction.

    p ∧ ¬p

    A compound proposition that is neither a tautology nor a contradiction is called a contingency.

    Another definition of equivalent:

    The compound propositions p and q are called logically equivalent if p ↔ q is a tautology. The notation p ≡ q denotes that p and q are logically equivalent.

    Ex. De Morgan Laws
    ¬(p ∧ q) ≡ ¬p ∨ ¬q
